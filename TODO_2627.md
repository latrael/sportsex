## Phase 1 — Ingestion · 3–4 days

- [x] **R1.1** Typed FPL client: `bootstrap-static`, `fixtures`, `event/{gw}/live`. Retry with backoff, a real User-Agent, and raw response retention so any parse can be replayed without refetching.
      Lives in `apps/web/src/lib/fpl/`: `types.ts` (zod schemas), `parse.ts` (pure parsers + `parseByEndpoint` replay), `client.ts` (`FplClient`).
      Design decisions worth knowing:
      - **Fetching and parsing are separate.** Every parser takes plain `unknown`, so a payload archived today can be re-parsed by tomorrow's code with no network call. The client returns `{ data, raw }` and takes an `onRaw` hook — point it at a table or blob store in R1.5 and the whole backfill becomes replayable.
      - **Every schema is `.passthrough()`.** FPL adds fields mid-season without notice (`defensive_contribution` is a 25/26 arrival). Unknown fields must never fail a parse.
      - **Decimal fields stay strings** (`"expected_goals": "8.23"`), exactly as the wire carries them. Parsing them to JS numbers here would put a float in front of every Decimal column; Prisma takes the string directly. This is R2.4's rule applied one layer earlier.
      - Retry covers 408/425/429/5xx, transport errors, and truncated bodies (a half-received 2 MB payload is a transport problem, not a schema problem). Full-jitter exponential backoff, `Retry-After` honoured up to 60s and a hard fail beyond that. 4xx other than those listed fail immediately. `random`, `sleep` and `fetch` are all injectable, so backoff is asserted exactly and no test spends real seconds.
      **Accept:** met. 39 unit tests, no network, against four checked-in payload fixtures in `src/test/fixtures/fpl/` — genuine trimmed 25/26 responses, referentially closed (every `element.team` resolves to a team in the file, every `explain[].fixture` to a fixture in `fixtures.sample.json`). GW33 is a real double gameweek, so the per-fixture attribution R1.4 depends on is already under test. Full suite 119 passed (80 → 119); typecheck and lint clean.
      *Verified separately against the live API* (not part of the suite): 20 teams, 841 players, 38 events, 380 fixtures, and GW1/26/33/38 live all parse — 79 players with 2+ fixtures in GW26, 248 in GW33.

      4. *Independent scorer check* — our top 20 scorers match football-data.org's on both name and goal count.
      5. *Coverage* — every fixture has ≥22 player rows with `minutes > 0`.
      **Accept:** all five pass and the results are written into this file. **If any check fails, we do not carry 25/26** — record which check failed and start clean at 26/27 GW1.

## Phase 3 — Valuation engine · 4–5 days

- [ ] **R3.1** Per-fixture performance score. Attack (goals, assists, xG, xA), defence (clean sheets, saves, tackles, interceptions, clearances, recoveries, xGC), discipline penalties, bonus. **z-scored within position group** so GK/DEF/MID/FWD land on comparable distributions.
      **Accept:** pure function, unit-tested, mean ≈ 0 and sd ≈ 1 within each position group across 25/26.

- [ ] **R3.2** Form: EWMA over recent fixtures, half-life ≈ 4 matches.
      **Accept:** unit test on a hand-computed sequence.

- [ ] **R3.3** Baseline with shrinkage: `(n₉₀·rate + k·cohort_prior) / (n₉₀ + k)`, k ≈ 6.
      **Accept:** a player with 90 minutes sits within 10% of the positional median; convergence to own rate is monotone in minutes.

- [ ] **R3.4** Availability multiplier from expected-minutes share (EWMA over last 6 fixtures, blended with starts rate) and FPL `status` / `chance_of_playing_next_round`.
      **Accept:** a player who stops appearing decays toward the floor over ~6 gameweeks; an `i`/`s` status player drops immediately.

- [ ] **R3.5** Compose `fairValue()`: `clamp(P_pos_base · exp(λ·(0.6·B + 0.4·F)) · M, 10, 1500)`.
      **Accept:** invariant 6 — pure, deterministic, no clock, no randomness.

- [ ] **R3.6** Backtest harness: replay 25/26 gameweek by gameweek, dump the price distribution at GW1/10/20/38.
      **Accept:** a reproducible report artifact, regenerable from one command.

- [ ] **R3.7** Calibration pass on λ, position weights, and half-life. This is the real work of the phase, not polish.
      **Accept:** all four position groups have players in the top 100 by price; the 5th–95th percentile price ratio is ≥10×; a manual sanity list (elite striker near the top, first-choice keeper above a fringe forward, injured player decaying) reads correctly.

- [ ] **R3.8** **Owner decision:** position weighting — what a clean sheet is worth relative to a goal. Determines whether this is a fantasy-football clone or its own thing. Decide deliberately; don't default to FPL scoring.
      **Accept:** decision recorded here with rationale, weights committed.
